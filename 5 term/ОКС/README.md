## lab2 - Учебный протокол с байт-стаффингом. Флаг: 0x7E в начале и конце.			
Написать коммуникационную программу, позволяющую формировать и передавать пакеты между двумя действительными либо условными пользовательскими станциями и реализующую один из 
алгоритмов корректного распознавания пакетов.
Структура пакета:										
- Start Flag 0x7E									
- Длина (1 байт)									
- Данные (N байт)									
- Контрольная сумма (1 байт, сумма по модулю 256)								
- End Flag 0x7E									
Метод распознавания: байт-стаффинг					

## lab3 - CRC как на бумаге														
Требования к интерфейсу:                                                                                                              
- Текстовое окно для пошагового отображения: текущих операций, обнаруженных ошибок, исправленных ошибок, результатов проверки
- Возможность вручную выбрать или добавить ошибку в конкретной позиции
- Подсветка ошибок — менять цвет ячеек или текста для ошибок и исправленных данных
- Статус отображения — например, статус проверки: ""Ошибки обнаружены"", ""Ошибки исправлены"", ""Данных без ошибок""
- Отдельное окно или панель, где выводится внутренняя логика (напримершаги алгоритма, промежуточные вычисления)
- Настройки длины блока, полинома CRC(хотя бы 2) 
- Дополнительно (по желанию). Кнопка ""Запустить пошагово"" — для пошагового выполнения алгоритма

## lab4 - Реализация токен-модели (Token Ring) с симуляцией передачи маркера											
											
Организовать топологию «кольца», где передача доступа осуществляется через передачу токена.											
В программе модель реализует передачу маркера по кольцу между станциями.											
Только одна станция может передавать сообщение, пока не передаст маркер следующей.											
В окна вывода — текущий владелец маркера, сообщения, время передачи.											
Возможно добавление функционала приостановки и повторной передачи при коллизиях или ошибках.											
Ввести механизм обнаружения отсутствия маркера (например, тайм-ауты).											
В случае потери маркера — реализовать процедуру восстановления.											
Имитация ошибок передачи (например, повреждение маркера или сообщения)		

## lab5 - Реализация юникаст- и бродкаст-адресации
Разработать коммуникационную программу, обеспечивающую передачу сообщений между несколькими пользовательскими станциями с использованием иерархической 
системы адресов. Программа должна реализовывать: 
- ввод и хранение таблицы станций с их адресами;
- передачу сообщений по указанному типу адресации (unicast, broadcast, multicast, 
anycast); 
- обработку полученных сообщений каждой станцией; 
- интерфейс управления передачей (ввод адреса назначения, выбор типа адресации, 
отправка сообщения); 
- отображение состояния сети (список станций, активные группы, принятые 
сообщения). 
Адреса должны иметь фиксированную длину и содержать поля для идентификатора уровня и номера станции (иерархическая структура). Формат сообщений включает 
служебные поля (тип адресации, адрес источника, адрес назначения, длина, данные).

- количество станций: 6
- опорная топология: шина 
- адресация: 8-битные адреса (0x00–0xFF), где 0xFF – бродкаст-адрес 
- реализовать: отправку сообщения по конкретному адресу (юникаст), отправку бродкаст-сообщения всем станциям сети, обработку входящих сообщений (станция выводит полученные данные),
интерфейс: поле ввода адреса назначения, поле ввода текста сообщения, кнопка «Отправить».
